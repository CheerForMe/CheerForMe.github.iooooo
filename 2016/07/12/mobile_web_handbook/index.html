运营商

运营商补贴

  • Web开发人员要记住的就是：通过决定卖给不知情的消费者哪种手机，运营商就能够影响移动浏览器市场，因为那些设备的内置浏览器将获得更多的市场份额。因此，跟踪运营商目前的偏好很重要。

第四章 视口

三个视口

布局视口

  • 在手机上，视口与移动端浏览器屏幕宽度不再相关联，而是完全独立的。我们称它为布局视口：CSS布局会根据它来计算，并被它约束。

  • 在手机上，为了容纳为桌面浏览器设计的网站，默认的布局视口宽度远大于屏幕的宽度。

视觉视口

  • 它是用户正在看到的网站的区域。用户可以通过缩放来操作视觉视口，同时不会影响布局视口，布局视口仍保持在原来的宽度。
  • 视觉视口与设备屏幕一样宽，并且它的CSS像素的数量会随着用户的缩放而改变。

理想视口

  • 它是对设备来说最理想的布局视口尺寸。
  • 显示在理想视口中的的网站拥有最理想的浏览和阅读宽度，用户刚进入页面时也不再需要缩放。
  • 只有当网站是为手机准备的时候才应该使用理想视口。
  • 只有当你主动地往页面里添加meta视口标签时理想视口才会生效。如果没有meta视口标签的声明，那么布局视口将会维持它的默认宽度。
  • 告诉浏览器：布局视口的宽度应该与理想视口的宽度一致。

哪一个视口

  1. 在桌面浏览器中，浏览器窗口就是约束你的CSS布局的视口（又称为初始包含块）。它也决定了用户可以看到什么。
  2. 在手机上，桌面视口被拆分成了两个：布局视口会限制你的CSS布局，视觉视口会决定用户看到什么。
  3. 移动浏览器还有一个理想视口，它是对于特定设备上的特定浏览器的布局视口的一个理想尺寸。
  4. 可以把布局视口的尺寸设置为理想视口。实际上，这就是响应式设计的基础。

解题

  1. position:fixed;
    答：定位相对于布局视口。（后续有作者答案时修改）

  2. vw和vh单位是相对视口的百分比：width:25vw意味着元素的宽度是视口宽度的25%。
    答：vw和vh相对于理想视口。（后续有作者答案时修改）

缩放

  • 缩放会影响桌面和手机的（视觉）视口的尺寸。
  • 放大使视觉视口变得更小，因为屏幕上显示的CSS像素更少了；
  • 缩小会让视觉视口更大，因为屏幕上的CSS像素更多了。
  • 因此缩放程度和视觉视口的大小是逆相关的：放得越大，视觉视口就越小。

桌面和手机的区别：手机上布局视口不会被缩放影响，但在桌面上布局视口和视觉视口是相同的，不可能在改变其中一个视口的时候不改变另外一个。

分辨率

物理分辨率

  • 所有屏幕都有物理分辨率。
  • 像素的数量 除以 屏幕的以英寸为单位的宽度 = 设备每英寸的点数（DPI）。
  • 每英寸内的像素数量越多越好，因为这意味着画面会更清晰。
  • JavaScript无法获取设备的物理尺寸。

设备像素比

  • JavaScript有window.devicePixelRatio
  • CSS有device-pixel-ratio（基于WebKit的浏览器）。
  • CSS还有分辨率（所有浏览器都支持）的媒体查询。
  • 以上代码都和物理像素无关。
  • 它们为你提供了设备像素个数和理想视口的比，称为设备像素比（Device Pixel Ratio，简称DPR）。
  • 早期iPhone的设备宽度是320个物理像素，理想视口的宽度也是320个像素，所以设备像素比（DPR）是1。
  • 后来iPhone的宽度变成640个设备像素，而他们的理想视口宽度仍为320个像素，因此DPR变成2。
  • DPR不一定是整数。

dppx和dpi

  • JavaScript中的window.devicePixelRatio和媒体查询的device-pixel-ratio的单位都是dppx。
  • dppx：每一个像素的点数。

  • 实际上，不允许在dppx后面添加单位。

  • 为了兼容IE11及以下版本浏览器，我们必须使用dpi单位来代替。

  • 一英寸对应了CSS中的96个像素，所以1dppx等于96dpi。

meta视口

  • meta视口标签存在的主要目的是让布局视口的尺寸和理想视口的尺寸匹配。
  • 由Apple发明，其他手机和平板复制了它的大部分内容。
  • 桌面浏览器不支持也不需要，因为它们没有理想视口的概念。

  • 书写方式：

  • 每一个名/值对都是一个给浏览器发号命令的指令：

    1. width：设置布局视口的宽度为特定的值。
    2. init-scale：设置页面的初始缩放程度和布局视口的宽度。
    3. minimum-scale：设置了最小缩放程度（用户可缩小的程度）。
    4. maximum-scale：设置了最大缩放程度（用户可放大的程度）。
    5. user-scalable：是否阻止用户进行缩放。

minimum-scale和maximum-scale

  • 设置缩放程度的最小值和最大值。
  • 没有这些指令的时候，浏览器允许用户的缩放级别达到5（20%~500%）。
  • 有这些指令的时候，范围可扩大到10（10%~1000%）。
  • 更高的缩放程度不被支持：maximum-scale=20与maximum-scale=10的效果一样。
  • 安卓WebKit不支持minimum-scale；它的缩放范围是4（25%~400%）并且不支持改变范围。
  • IE可能没有正确工作。
